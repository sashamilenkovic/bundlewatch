## Nuxt 3 / Nuxt 4 (Vite mode)

Nuxt exposes Vite plugins via `vite.plugins`. BundleWatch runs as part of `nuxi build`, capturing both SPA and SSG output.

> Example: `examples/nuxt-app`

```ts
// nuxt.config.ts
import { bundleWatch } from '@milencode/bundlewatch-vite-plugin';
import { defineNuxtConfig } from 'nuxt/config';

export default defineNuxtConfig({
  vite: {
    plugins: [
      bundleWatch({
        compareAgainst: 'main',
        saveToGit: process.env.CI === 'true',
        generateDashboard: true,
      }),
    ],
  },
});
```

**Notes**
- Nitro writes prerendered HTML into `.output/public`; BundleWatch still gets accurate sizes because the Vite hook receives in-memory chunks before Nitro emits files.
- Disable locally via `bundleWatch({ enabled: process.env.ANALYZE === 'true' })`.
- In CI add `pnpm --filter @bundlewatch/docs generate && pnpm build` if you also maintain the docs workspace inside the same repo.

## SvelteKit

Expose the Vite plugin through `kit.vite`. Works with any adapter (Node, Cloudflare, etc.) because BundleWatch executes during the Vite build phase.

> Example: `examples/sveltekit-app`

```ts
// svelte.config.js
import adapter from '@sveltejs/adapter-auto';
import { bundleWatch } from '@milencode/bundlewatch-vite-plugin';

const config = {
  kit: {
    adapter,
    vite: {
      plugins: [
        bundleWatch({
          compareAgainst: 'main',
          printReport: true,
          saveToGit: process.env.CI === 'true',
        }),
      ],
    },
  },
};

export default config;
```

**Tips**
- SvelteKit often outputs to `.svelte-kit/output/client`; BundleWatch logs the analyzed files in the console so you can confirm whatâ€™s tracked.
- When using adapters that rebuild during prerender (`adapter-static`), keep `bundleWatch` enabled only for production builds (`if (!process.env.DEV)`).

## SolidStart

SolidStart projects export a normal Vite config. Just append the plugin to the array.

> Example: `examples/solidstart-app`

```ts
// vite.config.ts
import { defineConfig } from 'vite';
import solid from 'solid-start/vite';
import { bundleWatch } from '@milencode/bundlewatch-vite-plugin';

export default defineConfig({
  plugins: [
    solid(),
    bundleWatch({
      saveToGit: process.env.CI === 'true',
      compareAgainst: 'main',
    }),
  ],
});
```

**Notes**
- Works for SSR (`solid-start-node`) and edge adapters because BundleWatch runs during the build step, not the runtime.

## Astro

Astro exposes Vite via the `vite` key in `astro.config`. This captures both client and server bundles generated by Astro integrations.

```ts
// astro.config.mjs
import { defineConfig } from 'astro/config';
import { bundleWatch } from '@milencode/bundlewatch-vite-plugin';

export default defineConfig({
  vite: {
    plugins: [
      bundleWatch({
        generateDashboard: true,
        failOnSizeIncrease: true,
        sizeIncreaseThreshold: 15,
      }),
    ],
  },
});
```

**Notes**
- Astro may emit `server/entry.mjs` and `client/*.js`; both show up in the BundleWatch treemap.
- Turn off dashboard generation locally if you only need console output: `generateDashboard: process.env.CI === 'true'`.

## Qwik City

Qwik apps also rely on Vite. Drop BundleWatch into the config used by `qwik build`.

```ts
// vite.config.ts
import { defineConfig } from 'vite';
import { qwikCity } from '@builder.io/qwik-city/vite';
import { bundleWatch } from '@milencode/bundlewatch-vite-plugin';

export default defineConfig(() => ({
  plugins: [
    qwikCity(),
    bundleWatch({
      compareAgainst: 'main',
      saveToGit: process.env.CI === 'true',
    }),
  ],
}));
```

Because Qwik ships lots of tiny chunks, enabling the parser-based Vite plugin (work in progress) will give more precise dependency graphs in the dashboard.

## Remix

Remix can be configured to build with Webpack or Vite depending on the adapter. Use the matching BundleWatch plugin:

### Remix + Webpack

```js
// remix.config.js or webpack.config.js
const { BundleWatchPlugin } = require('@milencode/bundlewatch-webpack-plugin');

module.exports = {
  // ...your webpack config
  plugins: [
    new BundleWatchPlugin({
      compareAgainst: 'main',
      generateDashboard: true,
    }),
  ],
};
```

### Remix + Vite

Follow the regular [Vite instructions](./vite) since Remix simply forwards to a Vite config.
